# What's new in Note Toolbar v1.32?

_Make sure you've updated to the latest version to get all of these changes._

This release addresses the majority of the issues raised on the new [Obsidian Community Scorecard ↗](https://community.obsidian.md/plugins/note-toolbar#scorecard). (CSS issues are a WIP.) See [#546](https://github.com/chrisgurney/obsidian-note-toolbar/issues/546).

## Improvements 🚀

- Updated [Ukrainian translations](https://github.com/chrisgurney/obsidian-note-toolbar/pull/548). _Thanks @laktiv!_

## Fixes 🛠️ 

- Canvas cards can now be connected when toolbars are in any position. [#415](https://github.com/chrisgurney/obsidian-note-toolbar/issues/415) _Thanks @kotaindah55!_
  - Note that there's an Obsidian bug on phones: When the canvas file is first opened, the connector is offset from where your finger's positioned, regardless of toolbar position.
    - Workaround: Disable Obsidian's _Appearance → Sliding sidebars_ option, and it's fixed. Or change the position of the toolbar while the canvas is open, for a temporary fix.
- Settings: You can now key down past break/separator/spreader items in a toolbar's item list.
- API: Floating toolbars via `ntb.toolbar` were being immediately removed after being triggered by a script; this is fixed.
- Command items: Tooltips for unavailable items were not being set back their initial state, once they were available again.
