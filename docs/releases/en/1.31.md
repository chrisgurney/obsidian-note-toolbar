## New Features 🎉

### Note Toolbar CLI

The [Note Toolbar CLI](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Note-Toolbar-CLI) provides command-line access to create toolbars and items, using [Obsidian's CLI ↗](https://obsidian.md/help/cli). This lets you control Note Toolbar from your terminal for scripting, automation, and integration with external tools.

> [!NOTE]
> Out of an abundance of caution, deletion of items and toolbars is currently not supported in the CLI. Perhaps such commands will be added if there is a clean way to undo these actions (for which currently there is not).

> [!IMPORTANT]
> Access to the CLI requires Obsidian 1.12 (and the installer for that version) or later.

```sh
# List toolbars
obsidian note-toolbar

# List items in a toolbar
obsidian note-toolbar toolbar="Daily Notes"

# Create a toolbar
obsidian note-toolbar:new name=Formatting

# Add an item to a toolbar
obsidian note-toolbar:add-command to=Formatting label=Bold command=editor:toggle-bold

# Use an item
obsidian note-toolbar:use item=c0a21325-b8b1-4617-81a0-5114350310fc
```

[Read the documentation](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Note-Toolbar-CLI) and _[let me know what you think!](https://github.com/chrisgurney/obsidian-note-toolbar/discussions/539)_

### Agent skills

Added `SKILL.md` files for agents that use them, to use the Note Toolbar CLI, write code using the Note Toolbar API, and to generate Note Toolbar Callouts.

> [!note]
> Note that no AI will ever be built into this plugin. Agent skills are just text specifications that allow for external AIs to interact with it more efficiently.

[Learn more](https://github.com/chrisgurney/obsidian-note-toolbar/tree/master/skills/README.md) and _[let me know what you think!](https://github.com/chrisgurney/obsidian-note-toolbar/discussions/539#discussioncomment-16907044)_

### Color callouts

Add color to Note Toolbar Callouts:

- Use these colors (built-in to Obsidian's standard styles), which should adapt to your theme:
  - `blue`, `cyan`, `green`, `orange`, `pink`, `purple`, `red`, `yellow`
- Add the word `icon` (before or after the color) to just color the icons (e.g., `icon-red`).

<img width="500" src="https://github.com/user-attachments/assets/e172a444-f44b-4d43-9d3a-710e7ce471df" />

Example:
```md
> [!note-toolbar|button-icon-red] Basic
> - [[Welcome To The Vault.md|:LiHome: Home]]
> - [:LiGlobe:](<https://chrisgurney.ca/>)
```

_Thank you Saorsa for [the suggestion](https://github.com/chrisgurney/obsidian-note-toolbar/discussions/544)!_

## Fixes 🛠️

- Settings (data.json): The `recentFiles`, `recentItems`, `recentToolbars` settings will be removed from `data.json`, if they exist. [#542](https://github.com/chrisgurney/obsidian-note-toolbar/issues/542) (They moved to local storage in a previous update.)

## API

- Improved: More robust handling of arguments provided to scripts.
- `ntb.setSelection` now places the provided text at the cursor position, if there's no selection or current word.
