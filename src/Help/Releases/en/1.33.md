# What's new in Note Toolbar v1.33?

_Make sure you've updated to the latest version to get all of these changes._

This release addresses the remainder of the issues raised on the new [Obsidian Community Scorecard ↗](https://community.obsidian.md/plugins/note-toolbar#scorecard). See [#546](https://github.com/chrisgurney/obsidian-note-toolbar/issues/546) for updates.

## Improvements 🚀

### Share individual toolbar items

> Available in `1.33.24`

Edit an item, go to ***more actions (...) → Share item...*** to get a URL that you can share with another user.
- See it in action in the [Additional Gallery items](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Additional-Gallery-items) page, where all the **Add this item** buttons were created from these shared links.

### Files in modals: Use originating file as context

> Available in `1.33.19`

<img width="600" src="https://github.com/user-attachments/assets/936c6967-be04-4a85-b324-0f5d514fa836" />

When opening files in modals, set the **Use originating file as context** option to specify that the active file that opened the modal should be used as context on rendering.
- For example, this allows you to open a Base file that shows backlinks in a modal, and have it show the backlinks in the file that _opened_ the modal.
- To specify a different file altogether to use as context, see the new `ntb.modal` API option, below.
- _Thank you @nefesh1-almagor1 for [the suggestion](https://github.com/chrisgurney/obsidian-note-toolbar/issues/402)!_

### Other improvements

- `1.33.20` Using **right-click → Swap toolbars...** or the API to set the `notetoolbar` property (via `ntb.setProperty`), now immediately updates the toolbar.

## Fixes 🛠️

- `1.33.25` When opening Bases in modals they showed truncated results. [#559](https://github.com/chrisgurney/obsidian-note-toolbar/issues/559) _Thank you @nefesh1-almagor1!_
- `1.33.25` In modals opened for files (or by the API), the close button was hard to click on as it was overlapped by content.
- `1.33.24` Gallery: Added back in missing Gallery items from the index (that are now in [Additional Gallery items](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Additional-Gallery-items)), and an outdated icon. 
- `1.33.24` Export: Comments were previously breaking exported scripts; they're now removed.
- `1.33.17` The source path was not being passed to embedded files in modals (i.e., File item + open in modal).

## Changes

To address a [Obsidian Community Scorecard ↗](https://community.obsidian.md/plugins/note-toolbar#scorecard) recommendation, usage of the clipboard has been removed across the plugin:
- Actions that previously used the clipboard (e.g., _right-click → Copy as callout_) now show a dialog you can copy the text from, and provide more context.
- Gallery: Items that used the clipboard have been moved to [Additional Gallery items](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Additional-Gallery-items); upon adding them, a prompt directs you to this page.
- `1.33.17` Release notes and Tips are now included in the plugin, instead of being loaded from an external domain.

## API

### `ntb.modal` context options

> Available in `1.33.19+`

When opening another file (such as a Base) with `ntb.modal`, you can now specify the file that should be used as context on rendering.
- For example, this allows you to open a Base file that shows backlinks in a modal, and have it show the backlinks in the file that _opened_ the modal.

To use the _active file_ as context for a Base:

```js
const base = "Bases/Backlinks.base";
const baseFile = ntb.app.vault.getAbstractFileByPath(fn);
if (baseFile) { 
  await ntb.modal(baseFile, {
    originAsContext: true
  });
}
```

Or to provide a _completely different file_ as context for the Base:
```js
...
  await ntb.modal(baseFile, {
    contextPath: "Other File.md"
  });
...
```

### ⚠️ Breaking changes

The API was changed to accomodate warnings provided in the Scorecard:

- API: `ntb.clipboard()` is deprecated and will no longer work. For a workaround, see [the documentation](https://github.com/chrisgurney/obsidian-note-toolbar/wiki/Note-Toolbar-API#deprecated).
- `1.33.15` API: `ntb.fileSuggester()` now requires a list of files to be provided as the first parameter:

```ts
const fileOrFolder = await ntb.fileSuggester(
  ntb.app.vault.getAllLoadedFiles()
);
if (fileOrFolder) new Notice(fileOrFolder.name);
```

```ts
// show only folders
const folder = await ntb.fileSuggester(
  ntb.app.vault.getAllLoadedFiles(),
  { foldersonly: true }
);
if (folder) new Notice(folder.name);
```
